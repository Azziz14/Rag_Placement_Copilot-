# Project Guide

You are building InterviewPilot AI.

## Development Rules & Principles

- **Build one module at a time**: Focus on single components sequentially.
- **Never generate the whole project at once**: Implement code incrementally.
- **Wait after every module**: Allow review, testing, and alignment before proceeding.
- **Follow clean architecture**: Maintain clear separation of concerns.
- **Keep backend modular**: Structured directory setup for API, core services, database models, etc.
- **Keep frontend modular**: Component-driven design with reusable hooks and utilities.
- **Use FastAPI**: High-performance python web framework for backend APIs.
- **Use Next.js**: React framework for production-grade frontend web apps.
- **Use PostgreSQL**: Primary relational database for user data, profiles, and interview sessions.
- **Use Qdrant**: Vector database for RAG context, parsing data, and similarity search.
- **Use LangChain**: Framework for constructing LLM-driven applications.
- **Use LangGraph**: Build stateful, multi-actor applications with LLMs.
- **Use Clerk Auth**: Complete authentication and user management solution.
- **Add comments**: Write descriptive code comments for complicated logic.
- **Add docstrings**: Document all functions, classes, and modules clearly.
- **Production-grade code only**: Clean, robust, well-typed, and secure code.
